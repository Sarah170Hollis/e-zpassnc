Contribution Guidelines

Thank you for your interest in contributing to e-zpassnc! We appreciate your time and effort to help improve this project. Please follow the guidelines below to ensure a smooth collaboration process.

How to Contribute

Fork the Repository – Click the "Fork" button on GitHub.

Clone Your Fork – Clone your personal fork to your local machine.

Create a New Branch – Use a clear and descriptive branch name.

Make Changes – Write clean, well-documented, and efficient code.

Test Your Changes – Ensure nothing is broken before submitting.

Submit a Pull Request – Provide a detailed explanation of your changes.

Reporting Issues

If you discover a bug, please follow these steps:

Search existing issues to avoid duplicates.

Provide clear steps to reproduce the bug.

Include screenshots or logs (if applicable).

Describe your environment (OS, browser, Node version, etc.).

Suggesting Features

We welcome new feature suggestions! Please:

Clearly describe the problem your feature solves.

Provide examples or use cases.

Suggest potential implementation ideas (optional).

Code of Conduct

All contributors must follow these basic rules:

Be respectful and inclusive.

Avoid offensive or discriminatory language.

Provide constructive and helpful feedback.

Collaborate openly and responsibly.
